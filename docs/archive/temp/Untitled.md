         ⚠️ 严重程度：中

         1. batch 模式缺盈利质量分（Layer 1 不完整）

         设计要求 Layer 1 满配 50 分，但 score_batch() line 183-253 的 _score_quality_batch 只处理了估值（PE/PB 排名，最高 25 分），完全缺失：

         - ROE 评分（设计 10 分）
         - 毛利率评分（设计 5 分）
         - 经营现金流评分（设计 5 分）
         - 资产负债率评分（设计 5 分）

         _quality_single() 有完整实现，但 score_batch() 不调用它。导致批量评分时，所有股票的 Layer 1 上限只有 25 分而非 50 分，Layer 1 实质上残缺。

         2. batch 模式缺资金流向评分（Layer 3 不完整）

         设计 Layer 3 中资金流向占 8 分（主力净流入占比 + 换手率健康度），但 _score_sentiment_batch() 只用了 spot 数据中的 turnover 和 pct_change，完全缺失
         main_net_inflow_pct。_sentiment_single() 有资金流评分，但 batch 模式不调用。

         3. KDJ 参数略有偏差

         # line 614: scan_industry("") 返回所有股票而非指定个股的行业
         stocks = screener.scan_industry("", top_n=1)  # ← bug：第一个参数是空字符串！

         candidate_sector = "sh" if sym.startswith(("6", "5", "9")) else "sz"

         5. score_batch() 最终合并逻辑有缺陷

         # line 106-108
         final = pd.concat(
             [df[df["vetoed"] == True], active], ignore_index=True
         ).sort_values("total", ascending=False, na_position="last")

         被否决的股票（vetoed == True）的 total 列为空（NaN），sort_values 会将 NaN 排在最后，但 vetoed == True 的 total 列实际上是 NaN 而非
         0——这意味着被否决股以 NaN 参与排序而非 0，可能在某些 pandas 版本导致排序不稳定。

       ---
         五、关键结论

         1. 设计文档 vs 实现：覆盖约 65%。Layer 2（技术面）实现最完整，Layer 1/3 在 batch 模式下严重缺失盈利质量和资金流向评分。
         2. simulation.py 的 scan_and_buy() 依赖 score_batch()，因此买入决策实际上是用残缺的评分体系（Layer 1 只有估值分）进行筛选，可能导致基本面优质但 PE
           排名不靠前的股票被低估。
         3. 最关键的 bug 是 B1（行业集中度惩罚失效）——在模拟环境中可能买入多只同板块股票，放大板块风险。
         4. score_single() 实现相对完整，用于卖出决策时各层评分齐全，但同样依赖未实现的 EventDriver。
