# MoatX 评分算法深度测评与提升方向

> 基于 `SCORING_ALGORITHM.md` v1 版本的独立评审
> 角色：站在对立面，找盲区，不找补

---

## 一、算法当前最大的问题

**当前设计本质：个股快照打分器。** 它独立评估每一只股票在一个时间点上的"好坏"，然后按分数排名买入。

**但投资不是在真空中挑好股票。** 投资要回答的是：

> "在我已经持有 A/B/C 三只股票的前提下，要不要买 D？买多少？D 和 A 有没有重叠风险？如果大盘下周暴跌，我的持仓扛得住吗？"

当前算法完全不考虑这些问题。这是结构性缺陷，不是参数调优能解决的。

---

## 二、十个盲区

### 盲区 1：组合层面零考量

**现象：** 5 只顶级股票可能全是同一个板块。霍尔木兹事件下 Layer 4 把石油板块×1.4，5 只全买石油股，油价一回落全崩。

**缺失：** 没有组合集中度检查。应该：
```
if 新股票所在行业 == 已有持仓中占比最大的行业:
    降低买入优先级（或要求更高分数才能入场）
if 新股票与已有持仓相关性 > 0.7:
    降低仓位权重
```

**数据源：** 行业板块已有（`screener.scan_industry`），相关性可从日线收益计算。

---

### 盲区 2：不同市场环境应该用不同权重

**现象：** 牛市中技术面（Layer 2）应该加权重（趋势更重要），熊市中质量面（Layer 1）应该加权重（便宜才是硬道理）。固定 50/35/15 是懒政。

**缺失：** 市值加权指数 20 日均线方向可作为市场状态的代理变量：
```
if 大盘 20 日均线上行（牛市）:
    Layer 2 权重 +10%, Layer 1 权重 -10%
if 大盘 20 日均线下行（熊市）:
    Layer 1 权重 +10%, Layer 2 权重 -10%
if 大盘横盘（震荡）:
    保持默认权重，加大 Layer 3（资金流向）权重
```

**数据源：** `StockData.get_daily("000300")` 沪深 300 指数。

---

### 盲区 3：没有反馈学习机制

**现象：** 算法给某只股打了 85 分，重仓买入，20 天后亏了 15%。算法不会因此调整自己的判断逻辑——下次同类股票还是 85 分。

**缺失：** 需要交易结果回灌到评分系统：
- 记录每笔交易：买入分数、各层得分、最终盈亏
- 定期回溯：哪些因子组合实际赚钱了？哪些总是亏？
- 自适应权重：亏损因子降权，盈利因子加权

这不需要复杂 ML。一张简单统计表就够了：
```
因子       买入次数   平均收益   胜率    →   调整方向
PE<10      15        +8.2%    73%         ✅ 保持高权重
KDJ超卖    23        -3.1%    34%         ⚠️ 降权
RSI超卖    18        +1.5%    56%         → 保持
```

---

### 盲区 4：只买不管理仓位

**现象：** 算法只有"买"和"卖"两个动作，没有"加仓"和"减仓"。一只股票从 60 分涨到 85 分（更强了），已经持有的仓位不会增加。

**缺失：** 
- 持仓股票的分数每天更新 → 分数上升 → 加仓，分数下降 → 减仓
- 浮盈加仓规则：盈利超过 5% 且分数仍 ≥ 70 → 追加仓位
- 浮亏减仓规则：亏损超过 5% 且分数降至 ≤ 40 → 减半仓

---

### 盲区 5：A 股独有信号完全缺失

**现象：** A 股是全球最特殊的市场，散户占比 60%+，政策驱动极强。当前算法用了一套通用的量化因子，完全没利用 A 股特有信号：

| A 股特有信号 | 含义 | 当前状态 |
|-------------|------|----------|
| 北向资金净流入 | 外资动向，A 股最重要的先行指标之一 | ❌ 未接入 |
| 融资融券余额变化 | 杠杆资金情绪 | ❌ 未接入 |
| 涨停板封单量 | 强势股识别（封单越大越强） | ❌ 未接入 |
| 龙虎榜机构席位 | 机构动向 | `stock_data.py` 有 `get_limit_up()` 但未接入评分 |
| 次新股/解禁日期 | 解禁前后有显著异常收益 | ❌ 未考虑 |
| 高送转/分红预案 | A 股特有的炒作题材 | ❌ 未接入 |

---

### 盲区 6：Layer 4（事件驱动）当前是纯关键词匹配

**现象：** "霍尔木兹" 命中 → 石油板块 +20 分。但不同性质的事件影响完全不同：
- 供给冲击（封锁海峡）→ 油价涨，航运涨，但化工下游跌
- 需求复苏（经济重启）→ 油价涨，航运涨，化工也涨
- 地缘冲突 → 油价涨，军工涨，但旅游航空跌

纯关键词匹配区分不了这些。同样"涨"，因果链完全相反。

**提升方向：** 
- 事件分类：供给冲击 / 需求变动 / 政策驱动 / 地缘冲突 / 技术突破
- 每类事件定义上游受益行业和下游受害行业
- 同一个事件下：受益行业加分，受害行业减分

---

### 盲区 7：缺少交易执行层面的考量

**现象：** 算法给一只小盘股打了 90 分，但它的日成交量只有 200 万。模拟买 5 万没问题，但实际上 5 万的买单就可能把股价推高 2%。流动性成本在模拟交易中不可见，在真实交易中很致命。

**缺失：**
- 流动性过滤：日成交额 < 5000 万的股票降权或过滤
- 冲击成本估算：计划买入金额 / 日均成交额 > 1% → 需要拆单或放弃

---

### 盲区 8：PE 排名法有隐蔽陷阱

**现象：** PE 排名法是好的（跨行业可比），但有一个坑：

负 PE 的股票在 `rank(pct=True)` 中排在哪里？如果把它们排在最前面（因为是负数，最小），那亏损最严重的股票拿到了最高的估值分。

**需要处理：** PE ≤ 0 的股票，估值分强制 = 0（PE 为负，估值无意义，只允许技术面交易）。

---

### 盲区 9：没有最大回撤控制

**现象：** 算法独立评分每只股票，但没有全局回撤控制。如果模拟账户已经亏了 15%，今天开盘还是照常买入——它不知道自己在"抄底"还是"接飞刀"。

**缺失：**
- 账户级回撤熔断：累计亏损 > 15% → 暂停新买入，只持有或卖出
- 单股回撤熔断：某只股票从高点回落 > 10% → 不再加仓

---

### 盲区 10：多空不对等

**现象：** 买入需要 4 层评分全通过，但卖出只靠 4 条机械规则（止盈/止损/技术/超期）。卖出信号的质量远低于买入信号。

**应该：** 每天重新评分持仓股票。分数骤降（例如从 75 降到 40）比"止损 -7%"更能说明问题——也许不是市场波动，是基本面真的恶化了。

---

## 三、优先级评估

| 优先级 | 盲区 | 理由 |
|--------|------|------|
| P0 | 盲区 1：组合集中度 | 不改可能导致 5 只全买石油股 |
| P0 | 盲区 2：市场环境权重 | 简单实现，收益大 |
| P0 | 盲区 8：负 PE 处理 | 不改会导致估值分完全错误 |
| P1 | 盲区 10：多空不对等 | 卖出和买入应该用同一套标准 |
| P1 | 盲区 4：仓位管理 | 浮盈加仓/浮亏减仓是收益增强的关键 |
| P1 | 盲区 5：A 股特有信号 | 北向资金/龙虎榜/融资余额是最强 A 股因子 |
| P2 | 盲区 6：事件分类升级 | 需要人工维护分类表 |
| P2 | 盲区 3：反馈学习 | 需要积累数据后才能启用 |
| P3 | 盲区 7：流动性过滤 | 模拟交易下影响小，实盘前必须做 |
| P3 | 盲区 9：回撤熔断 | 简单但实际触发频率低 |
