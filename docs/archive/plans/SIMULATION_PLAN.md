# MoatX 模拟交易阶段计划

> 目标：全自动模拟基金经理 — 交易日自主选股、自主买卖、自主复盘

---

## 一、核心流程

```
09:30  开盘扫描 → 全市场选股 → 模拟买入候选
09:30-15:00  盘中监控 → 持仓预警 + 卖出信号检测
15:00  收盘执行 → 卖出信号触发模拟卖出
15:10  账户快照 → 记录当日模拟账户完整状态
15:30  每日报告 → 持仓变动、盈亏、预警汇总
```

## 二、改造内容

### 2.1 候选股自动模拟买入

**当前问题：** `import_candidates.py` 只导入到 candidates 表，不触发模拟交易

**改造内容：**
- 开盘扫描后自动计算每只候选股买入数量和金额
- 调用 `PaperTrader` 执行模拟买入
- 记录买入原因（PE/技术指标/风险分）
- 单票仓位上限（如 20%），总仓位上限（如 80%）

**涉及文件：** `scripts/import_candidates.py`, `modules/signal/paper_trader.py`

**验收标准：** 开盘后自动买入符合条件的候选股，模拟账户可见持仓

---

### 2.2 持仓卖出信号生成

**当前问题：** 信号引擎生成买入信号为主，缺少基于持仓成本的卖出逻辑

**改造内容：**
- 对 paper_holdings 中每只股票：
  - 止盈：浮盈超过 N% 卖出
  - 止损：浮亏超过 M% 卖出
  - 技术卖出：MACD 死叉 / KDJ 超买 / RSI > 70
  - 持有天数超限：超过 X 天未达预期卖出
- 信号写入 signal_journal，自动触发模拟卖出

**涉及文件：** `modules/signal/engine.py`, `modules/strategy/library.py`

**验收标准：** 持仓股票满足卖出条件时自动生成卖出信号并执行

---

### 2.3 调度器重构为模拟交易流程

**当前问题：** 现有 7 个任务面向"候选股验证"场景，不适合模拟交易

**改造内容：**

| 时间 | 任务 | 说明 |
|------|------|------|
| 09:30 | `scan_and_buy` | 全市场扫描 → 筛选候选 → 模拟买入 |
| 09:35 | `monitor_holdings` | 首次预警检查（之后每 30 分钟） |
| 14:55 | `generate_sell_signals` | 扫描持仓生成卖出信号 |
| 15:00 | `execute_signals` | 执行当日所有买卖信号 |
| 15:10 | `daily_snapshot` | 记录模拟账户快照 |
| 15:30 | `daily_report` | 生成当日完整报告 |

**涉及文件：** `modules/scheduler.py`

**验收标准：** 交易日无人值守自动运行完整流程

---

### 2.4 每日报告增强

**当前问题：** `daily_report.py` 只输出任务执行状态，缺少投资决策内容

**改造内容：**
- 新增模拟账户日报：
  - 今日操作：买入/卖出明细
  - 持仓变动：新增/清仓
  - 盈亏统计：当日盈亏、累计盈亏
  - 预警汇总：触发了哪些预警
  - 明日关注：候选股推荐
- 输出 Markdown + 飞书卡片格式

**涉及文件：** `scripts/daily_report.py`

**验收标准：** 每日收盘后生成完整投资日报

---

### 2.5 模拟交易参数配置化

**当前问题：** 买入/卖出参数硬编码在代码中

**改造内容：**
- `data/simulation.toml` 配置文件：

```toml
[buy]
max_single_pct = 0.2       # 单票仓位上限 20%
max_total_pct = 0.8        # 总仓位上限 80%
min_buy_score = 3          # 最低买入评分
pe_max = 50                # PE 上限
risk_score_max = 30        # 风险分上限

[sell]
stop_profit_pct = 0.15     # 止盈 15%
stop_loss_pct = 0.07       # 止损 7%
max_hold_days = 20         # 最长持仓天数
kdj_overbought = 85        # KDJ 超买卖出阈值
rsi_overbought = 75        # RSI 超买卖出阈值
```

**涉及文件：** `modules/config.py`, `data/simulation.toml`

**验收标准：** 修改配置文件无需改代码即可调整模拟策略

## 三、改造清单总表

| # | 改造点 | 复杂度 |
|---|--------|--------|
| 2.1 | 候选股自动模拟买入 | 中 |
| 2.2 | 持仓卖出信号生成 | 高 |
| 2.3 | 调度器重构为模拟交易流程 | 中 |
| 2.4 | 每日报告增强 | 中 |
| 2.5 | 模拟交易参数配置化 | 低 |

---

## 四、不做的

- 不接入真实券商 API
- 不动真实持仓数据（实盘 DB 只读）
- 不引入强化学习/复杂 ML 模型
- 不做高频/日内交易
- 模拟交易不碰 `portfolio.db` 的 holdings/trades 表，只操作 warehouse.db 的 paper_* 表

---

## 五、验收标准

- 一个交易日无人值守，自动完成：选股 → 买入 → 监控 → 卖出 → 报告
- 模拟账户资金曲线可追溯
- 每日报告可飞书推送
- 所有参数可通过 TOML 文件调整
